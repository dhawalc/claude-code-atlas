# Claude Opus Max Prompt: Atlas Infographic System

Use this prompt in Claude.ai Opus Max.

---

You are an elite editorial information designer, product archaeologist, and frontend storytelling director.

Your task is to create a **jaw-dropping infographic system** based on the public Atlas for the `claude-code` codebase.

Primary goal:
Turn the research into something that feels like **"holy shit, I have never seen a software codebase explained like this before."**

This is **not** a generic docs task.
This is **not** a SaaS landing page task.
This is **not** a bland architecture summary.

This should feel like a mix of:
- high-end editorial feature
- systems x-ray
- product archaeology report
- beautifully designed architecture atlas
- internet-native explorable artifact

## Core Thesis

This codebase should be framed as:

**A terminal-native agent operating system whose deepest lessons are hidden in its safety model, control plane, extensibility substrate, and leaked product futures.**

## The 3 Editorial Lenses

Organize the output around these three lenses:

1. **Lessons**
   What this codebase teaches the world about building real agent systems.

2. **Invisible Inventions**
   The buried patterns, weird subsystems, and design moves most people would never notice on a casual read.

3. **Resolved Tensions**
   The difficult tradeoffs the codebase is actively negotiating:
   - autonomy vs control
   - local vs remote
   - platform vs product
   - speed vs safety
   - single-agent simplicity vs multi-agent orchestration

## Non-Negotiable Story Beats

Your infographic system must make these ideas legible and memorable:

- This is not “just a CLI”
- Safety is compositional, not a single guardrail
- Humans and models are different security principals
- Worktrees are a missing primitive for safe parallel agent coding
- Subagents are a control-plane problem, not just a prompting trick
- MCP/plugins/skills/bridge layers make this an ecosystem, not a single app
- The codebase leaks multiple product futures at once
- Memory is treated like an operating concern, not just extra context
- The beauty of the system is in the tensions it resolves, not just code cleanliness

## Creative Direction

Aim for a visual result that feels:
- cinematic
- technical
- editorial
- internet-native
- slightly dangerous
- sophisticated, not childish
- opinionated, not corporate

Avoid:
- generic developer gradients
- boring glassmorphism templates
- default SaaS dashboard aesthetics
- fluffy AI marketing language
- purple-on-dark cliché visuals

Preferred aesthetic cues:
- brutal editorial typography
- blueprint + dossier + control-room energy
- restrained but striking color system
- layered diagrams that reward close inspection
- poster-quality hero sections
- visual rhythm between dense analysis and bold statements

## Deliverables

Produce the output in 2 layers.

### Layer 1: Infographic Strategy
Provide:
- a strong title for the overall artifact
- a one-sentence thesis
- the top 5 “wow” moments
- the best narrative flow for a reader
- the most viral or quotable lines to highlight
- a visual system recommendation
- a content structure for a flagship infographic or mini-site

### Layer 2: Production Assets
Then generate either:

#### Option A: GitHub Pages / Web Output
If the material is better as a web artifact, generate:
- a page-by-page content map
- exact section names
- copy for each section
- diagram specs
- chart specs
- infographic callouts
- interactions/animations if useful
- HTML/CSS/JS or React-friendly markup where appropriate
- Mermaid diagrams where useful
- SVG concepts described clearly enough for implementation

#### Option B: Markdown / Static Artifact Output
If the material is better as content-first assets, generate:
- a set of Markdown files
- each file with a clear title and purpose
- diagram sections using Mermaid when possible
- infographic panel descriptions
- caption text
- poster copy
- social-card copy
- pull quotes
- a final “reader journey” sequence

You may choose one format or provide both if that strengthens the result.

## Important Constraint

Do not just summarize the site.
Your job is to **transform the research into a category-defining visual artifact**.

You are allowed to:
- reframe the narrative
- reorganize the material
- elevate buried discoveries
- invent better diagram structures
- sharpen the arguments
- propose surprising ways to visualize the ideas

But do not fabricate technical claims that are not grounded in the Atlas material.

## Specific Things I Want You To Invent

Please explicitly include:
- 1 flagship “master map” concept that explains the whole system
- 1 safety infographic concept
- 1 control-plane infographic concept
- 1 “product archaeology” infographic concept
- 1 “invisible inventions” poster concept
- 1 “resolved tensions” visual framework
- 1 social-media-friendly compressed infographic version
- 1 downloadable PDF/report concept

Respond in this structure:

1. **Creative Thesis**
2. **Best Format Choice**
3. **Flagship Artifact Concept**
4. **Visual System**
5. **Narrative Flow**
6. **Infographic Concepts**

11. **What Would Make People Say “WTF”**

## Extra Push

I do not want “good.”
I want something that makes a very online technical audience feel like they just discovered a hidden blueprint for the future of agent systems.

Be ambitious. Be sharp. Be specific. Make it beautiful.
